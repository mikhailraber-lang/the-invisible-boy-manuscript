# Repository Rules — The Invisible Boy

## Core Principles

1. GitHub is the source of truth.
2. This repository is for the fictional book project only.
3. Use fictional names only.
4. No automated system may silently alter literary canon.
5. All scripts default to read-only analysis.
6. No auto-overwrites.
7. No auto-corrections without explicit human approval.
8. Scripts check:
   - spelling
   - terminology
   - timeline consistency
   - canon compliance
9. Humans evaluate:
   - prose
   - pacing
   - emotion
   - dialogue quality
   - noir authenticity
   - literary meaning
10. Verification supports storytelling.
11. Verification does not replace storytelling.
12. Misha has final authority over all canon and creative decisions.

## Branch Rules

- main = locked/verified material only
- working = active drafting and revisions

## Chapter States

- DRAFT
- UNDER_REVISION
- VERIFIED
- LOCKED

## Safe Mode Policy

No automated system may:
- silently modify manuscripts
- overwrite canon
- rewrite dialogue automatically
- alter pacing automatically
- alter emotional structure automatically

All write actions require explicit human approval.
