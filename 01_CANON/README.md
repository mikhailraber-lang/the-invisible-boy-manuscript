# 01_CANON

This folder contains all canon-critical information for *The Invisible Boy*.

Materials in this folder define the authoritative narrative continuity of the project.

---

# Contents May Include

- canon rules
- character profiles
- timeline master
- terminology locks
- location references
- continuity references
- relationship constraints
- investigative case continuity
- noir authenticity safeguards

---

# Canon Purpose

The purpose of this folder is to preserve:

- narrative continuity
- terminology consistency
- character integrity
- timeline stability
- investigative coherence
- emotional continuity

Canon materials are considered repository-critical reference systems.

---

# Repository Rules

Nothing in this folder should be modified casually.

Changes to canon materials should undergo:

- continuity review
- reconciliation review if necessary
- verification review
- explicit author approval

Hard canon conflicts must be resolved before material moves into LOCKED.

---

# Canon Protection Principles

Verification systems may assist:

- terminology enforcement
- timeline validation
- continuity comparison
- canon conflict detection

Verification systems may NOT:

- rewrite canon automatically
- reinterpret author intent
- replace literary judgment
- override emotional continuity decisions

GitHub serves as the repository source of truth.

---

# Canon States

Canon materials may exist in several states:

- DRAFT → under development
- VERIFIED → continuity checked
- APPROVED → author approved
- LOCKED → immutable without formal revision

Only LOCKED canon is considered fully authoritative repository truth.

---

# AI Interaction Rules

AI systems may:

- reference canon
- compare canon
- validate continuity
- detect contradictions
- suggest reconciliations

AI systems may NOT:

- silently alter canon
- overwrite locked material
- reinterpret approved narrative intent
- introduce unauthorized terminology drift
- mutate timeline anchors automatically

---

# Conflict Resolution

If canon conflicts are detected:

1. Conflict must be documented
2. Contradicting materials must be identified
3. Reconciliation review must occur
4. Author authority determines final resolution
5. Locked canon supersedes drafts

---

# Canonical Priority Order

When contradictions occur, repository authority resolves in this order:

1. LOCKED canon
2. APPROVED canon
3. VERIFIED materials
4. Draft manuscripts
5. Experimental AI outputs
