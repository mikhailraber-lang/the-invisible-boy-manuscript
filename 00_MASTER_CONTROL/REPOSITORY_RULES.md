# REPOSITORY_RULES

## Core Principles

1. GitHub is the repository source of truth.
2. This repository contains fictional literary material only.
3. Use fictional names only.
4. No automated system may silently alter literary canon.
5. All scripts default to read-only analysis unless explicitly approved otherwise.
6. No auto-overwrites.
7. No automated corrections without explicit human approval.
8. Verification supports storytelling.
9. Verification does not replace storytelling.
10. Literary judgment remains human-directed.

# Safe Mode Policy

No automated system may:

- silently modify manuscripts
- overwrite canon
- rewrite dialogue automatically
- alter pacing automatically
- alter emotional structure automatically
- modify locked chapters without approval

All write actions require explicit human approval.

---

# Canon Protection Rules

Locked canon takes priority over workflow convenience.

Hard canon violations must be reviewed before merge into LOCKED.

Git history serves as the authoritative revision record.

---

# Human Authority

Final creative and canon authority belongs to Misha.

Verification systems are advisory unless:
- hard canon conflicts occur
- timeline impossibilities occur
- locked terminology violations occur

---

# Repository Philosophy

Mechanical consistency may be automated.

Literary authenticity may not be automated.

The purpose of this repository is to preserve:
- narrative continuity
- canon integrity
- noir authenticity
- emotional realism
- creative intent
